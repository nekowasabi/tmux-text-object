# Product Overview: tmux-text-object

## Product Description

tmux-text-object は、tmux の copy-mode-vi に Vim のようなテキストオブジェクト機能を提供する tmux プラグインです。Vim ユーザーが慣れ親しんだ `yiw`、`yi"`、`ya(` などの操作を tmux のコピーモードで実現し、効率的なテキスト操作を可能にします。

## Core Features

### Word/WORD テキストオブジェクト
- **iw (inner word)**: 単語（英数字とアンダースコア）をヤンク
- **aw (around word)**: 単語と周囲の空白をヤンク
- **iW (inner WORD)**: WORD（非空白文字）をヤンク
- **aW (around WORD)**: WORD と周囲の空白をヤンク

### Quote テキストオブジェクト
- **i"** / **a"**: ダブルクォート内/ダブルクォートを含む範囲をヤンク
- **i'** / **a'**: シングルクォート内/シングルクォートを含む範囲をヤンク
- **i`** / **a`**: バッククォート内/バッククォートを含む範囲をヤンク

### Bracket テキストオブジェクト
- **i(** / **a(**: 丸括弧内/丸括弧を含む範囲をヤンク
- **i[** / **a[**: 角括弧内/角括弧を含む範囲をヤンク
- **i{** / **a{**: 波括弧内/波括弧を含む範囲をヤンク
- **i<** / **a<**: 山括弧内/山括弧を含む範囲をヤンク

### Additional Features
- ヤンク後に自動的に copy-mode を終了（Vim の `y` オペレータと同様）
- クロスプラットフォームのクリップボード対応（pbcopy、clip.exe、xclip、wl-copy）
- ノーマルモード（テキストオブジェクトあり）とビジュアルモード（標準ヤンク）の両方で動作
- カーソル移動なし - 現在行から直接テキストを抽出
- tmux バッファとシステムクリップボードの両方に同時コピー

## Target Use Case

### Primary Users
- **Vim ユーザー**: Vim の操作に慣れ親しんだ開発者やシステム管理者
- **tmux ユーザー**: tmux を日常的に使用するターミナルヘビーユーザー
- **効率重視のユーザー**: キーボード操作のみでテキストを素早くコピーしたいユーザー

### Specific Scenarios
1. **コードレビュー**: tmux でログやコードを確認しながら、特定の関数名や文字列を素早くコピー
2. **ターミナル作業**: コマンド出力からパスやエラーメッセージを効率的に抽出
3. **ペアプログラミング**: tmux のペアセッション中に、クォートやブラケット内のテキストを素早く共有
4. **ログ解析**: JSON や構造化されたログから、特定のフィールド値を抽出

## Key Value Proposition

### Unique Benefits
1. **Vim ライクな操作感**: 新しい操作を覚える必要なく、既存の Vim スキルを tmux でも活用
2. **カーソル移動不要**: テキストオブジェクトを直接選択・ヤンクするため、視覚的なフィードバックがクリーンで高速
3. **TPM 対応**: Tmux Plugin Manager (TPM) で簡単にインストール可能
4. **シンプルな実装**: 軽量で依存関係が少なく、bash と tmux のみで動作

### Differentiators
- **既存プラグインとの違い**: 多くの tmux プラグインはマウス操作や複雑なキーバインディングを必要とするが、本プラグインは Vim ユーザーの筋肉記憶を活用
- **純粋なテキストオブジェクト実装**: Vim の `text-object` の概念を忠実に再現し、`inner` と `around` の区別を正確に実装
- **即座のクリップボード統合**: tmux バッファだけでなく、システムクリップボードにも自動的にコピー

## Current Status

### Implemented Features
- Word/WORD テキストオブジェクト（完全実装済み）
- Quote テキストオブジェクト（完全実装済み）
- Bracket テキストオブジェクト（完全実装済み）
- クロスプラットフォームクリップボード対応

### Known Limitations
- **単一行のみ**: テキストオブジェクトは現在行内でのみ動作（複数行非対応）
- **シンプルマッチング**: クォートとブラケットのマッチングは基本的なアルゴリズムで、ネスト非対応
- **エスケープ非対応**: エスケープされたクォート（`\"`など）は特別扱いされない
- **カーソル位置要件**: カーソルがターゲットのテキストオブジェクト上/内に位置する必要がある

### Future Enhancement Opportunities
- ネスト対応のブラケットマッチング
- エスケープシーケンスの処理
- 複数行テキストオブジェクト対応
- カスタムテキストオブジェクトの定義機能
